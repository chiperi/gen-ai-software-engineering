# knowledge/

Domain knowledge for humans and AI agents: reference, glossary, business rules — the things that
are **not** in the code and must not be guessed.

- [domain-glossary.md](domain-glossary.md) — canonical terms (PAN, network token, last4, limits, states…).
- [domain-business-rules.md](domain-business-rules.md) — the lifecycle state machine and `BR-xxx` rules that requirements cite.

These are the source of truth for terminology and invariants; `product/specs/` and `architecture/`
reference them rather than restating them.
